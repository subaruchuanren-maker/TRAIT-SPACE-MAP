# 起動手順

2つのターミナルを開いて、それぞれ以下を実行するだけ。

---

## ターミナル 1 ── バックエンド（FastAPI）

```powershell
cd C:\Users\waiam\Downloads\4-program\Trait-Space-Map\backend
..\.venv\Scripts\python.exe -m pip install -r requirements.txt
..\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --port 8000
```

> 初回だけ `pip install` が必要。2回目以降は最後の行だけでOK。  
> 起動後 → http://localhost:8000/docs でAPI一覧が見られる

---

## ターミナル 2 ── フロントエンド（React）

```powershell
cd C:\Users\waiam\Downloads\4-program\Trait-Space-Map\frontend
npm.cmd install
npm.cmd run dev
```

> 初回だけ `npm.cmd install` が必要。2回目以降は最後の行だけでOK。  
> 起動後 → http://localhost:5173 でアプリが開く

---

## 止め方

各ターミナルで `Ctrl + C` を押す。

---

## npm.cmd を使う理由

PowerShell のスクリプト実行ポリシーの制限で `npm` 単体だと弾かれる場合がある。  
`npm.cmd` はその制限を受けないので、こちらを使う。

> 毎回 `.cmd` を付けるのが面倒なら、PowerShell に以下を1回だけ実行しておくと通常の `npm` が使えるようになる。
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
> ```

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| `npm : スクリプトの実行が無効` | 実行ポリシー | `npm.cmd` を使う |
| `python.exe が見つからない` | venv がない | `.venv` フォルダが存在するか確認 |
| `Cannot connect to localhost:8000` | バックエンドが起動していない | ターミナル1を先に起動する |
| `localhost:5173 が開かない` | フロントが起動していない | ターミナル2を起動する |
